# Architecture blueprints (README-only design packs) for fullstack agent workflows.

| Example | Stack |
| --- | --- |
| react-nextjs-nestjs-monorepo | Next.js UI + NestJS API + shared contracts |
| saas-multi-tenant-bff | BFF + auth + tenant-aware APIs |
| payments-webhook-platform | Billing service + idempotent webhook processing |
| aws-edge-fullstack-delivery | CloudFront, ALB, API gateway, observability |
| aws-lambda-api-cloudfront-spa | AWS serverless SPA + API pattern |
| azure-apim-functions-static-web-app | Azure APIM + Functions + Static Web Apps |
| gcp-cloud-run-pubsub-nextjs | GCP Cloud Run + Pub/Sub + Next.js |
| java-spring-boot-ai-agent | Spring Boot + Spring AI + MCP agent tools |
| java-quarkus-kubernetes-native | Quarkus on Kubernetes with native image option |
| java-vaadin-fullstack-admin | Vaadin Flow admin UI + Spring Boot services |

Each example is an architecture blueprint with routing notes and validation guidance. These are not runnable scaffolds; use them to choose skills, presets, and proof paths before implementation.

## Skill pack examples

Stack-specific skills also include `examples/` (good vs bad) and `templates/` folders — same pattern as Spring Boot imported skills. See any folder under `skill-packs/`.
